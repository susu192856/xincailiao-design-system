# 新材道设计规范决策记录

本文件记录会影响网页、React、Vue、Figma 和 AI Skill 的正式设计决策。单个页面问题、个人偏好和未经确认的截图不能直接成为全局规则。

## 权威顺序

1. 设计负责人确认的规范决策。
2. `src/styles/tokens.css` 中的结构化 Token。
3. `figma/components.manifest.json` 中的组件合同。
4. 已验收的 React / Vue 实现。
5. 已验收的 Figma 组件和业务示例。
6. Review 案例；仅作为证据，不自动成为规则。

## DEC-001：规范范围

- 当前正式范围为 32 个组件 + 7 个基础规范。
- Alert、Dropdown、Skeleton 等可作为未来组件扩展项评估，但不计入当前范围。

## DEC-002：组件属性语义

- `variant` 表示结构或视觉层级。
- `tone` 表示业务颜色语义。
- `size` 表示尺寸。
- 交互状态优先使用原生或明确的状态属性，例如 `disabled`、`loading`、`open`、`selected`。
- 迁移期允许旧字段作为兼容别名，但文档和新代码只使用 canonical 字段。

## DEC-003：圆角

- 按钮、输入框、选择器、标签、分页等高频交互控件默认使用 `--radius-sm`（2px）。
- 卡片、描述列表和分组容器可使用 2–4px。
- 弹窗、抽屉使用 4–8px；后台优先 4px。
- 官网展示卡片可使用 8–16px，但必须明确为品牌展示场景。

## DEC-004：焦点

- 键盘焦点统一使用 `--focus-ring-*` Token。
- 默认焦点环使用 `--neutral-900`，产品蓝用于功能色和链接，不作为所有控件的统一焦点色。
- 所有可交互组件必须提供可见 `focus-visible`。

## DEC-005：主操作

- 每个独立操作区域最多一个主操作，不限制整个页面只能有一个 solid 按钮。
- 同一按钮组不混用多个不同 tone 的主按钮。
- 取消或返回在左，确认或提交在右。
- `task` 表示任务推进，使用黑色，适用于提交、确认、发布、创建等改变业务流程的操作。
- `product` 表示产品能力，使用蓝色，适用于分析、生成、连接、筛选、导出等可重复或可撤回的功能调用。
- `brand` 表示品牌转化，使用品牌红，适用于官网、门户营销、预约演示和立即体验等关键转化，不进入常规后台操作组。
- task、product、brand 构成三色三角：三者是平级主操作语义，按页面类型和操作意图选择，不互相充当主次关系。
- 同一操作组禁止 task、product、brand 中任意两个 solid 并列；若 task 已是主操作，product 必须降为 outline 或 text，brand 必须移出该操作组。
- `neutral + solid` 仅作为旧代码兼容，不进入新文档与 Figma 合同。

## DEC-006：Token 与硬编码

- 业务组件和页面不得硬编码设计值。
- Token 源文件、颜色预览和自包含 SVG 示例可以声明原始数值，但必须可追溯到 Token。
- 组件尺寸、动效、焦点、遮罩和响应式断点均纳入 Token。

## DEC-007：规范变更流程

1. 确认权威来源和影响范围。
2. 更新 Token 或 manifest。
3. 同步 React、Vue、网页、Markdown、Figma 和 Skill。
4. 增加或更新 eval / verification。
5. 运行 `npm run verify` 并人工检查代表页面。

## DEC-008：数据可视化

- 数据色与成功、警告、错误、信息提示状态语义色是两套独立合同，色相相近也不得互相替代。
- 数据色参考社区数据可视化资源的色系组织、组合数量和图表拆解方法；色值以当前颜色页同步的数据色板为准。
- 图表页作为数据与内容类规范入口存在，负责承载数据色组合规则、图表类型样式、标题表头拆解和可复用示例。
- 图表设计按“业务问题 → 数据关系 → 图表类型 → 色彩组合 → 标题/图例/坐标/表格拆解 → 边界检查”的顺序决策。
- 单系列图表优先单色，多系列才启用分类色；颜色必须配合标签、线型、点型或数值，不能成为唯一识别方式。
- 柱状图从零基线开始，不使用立体图；环图不超过 5 类，更多分类合并为“其他”并补充明细表。
- 当前图表页覆盖图表元素、柱状、堆叠柱、瀑布、直方、折线、面积、堆叠面积、环图、地图和开高低收等样式规则；复杂图表在移动端允许横向滚动。

## DEC-009：语义色文字与填充

- `success/warning/error-*-text` 服务于浅背景文字、图标和描边，需要较深色阶保证可读性。
- 实心语义按钮使用独立的 `*-solid`、`*-solid-hover`、`*-solid-active`，保持清晰、现代的大面积色彩表现。
- 品牌红与 danger 红继续保持不同 Token 与用途，不能因视觉接近而合并。
