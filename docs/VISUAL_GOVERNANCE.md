# 视觉治理与验收流程

本文件用于设计师在 Codex 中维护 2026 新材道设计规范。目标是把视觉问题转成可复用规则、可执行检查和可截图验收的流程，而不是反复做无边界的全盘检查。

## 维护顺序

1. 先稳定网页规范站，再同步 Figma。
2. 先修首页、布局、颜色、字体、间距、圆角、阴影等基础页面。
3. 再修 29 个组件页。
4. 每次只处理一个页面组或一个问题类型，例如标题标记、表格字号、卡片间距。
5. 29 个组件页没有完成视觉签核前，不进行 Figma 组件同步。

## Codex 修改要求

1. 先读取 `skills/xincailiao-design-spec/SKILL.md` 和对应 reference。
2. 页面标题、章节标题、说明卡片、规范表格优先使用 `src/components/docs` 中的共享组件。
3. 说明型表格必须使用 `DocsTable`，不能在页面里裸写 `<table>`。
4. 标题前标记必须使用 `SectionHeading` 或 `SubsectionHeading`，不能在页面里临时写红色或蓝色小方块。
5. 修改后运行 `npm run verify`；视觉修改还要提供桌面和移动端截图。

## 设计师验收清单

- 桌面 1440px：标题标记、卡片边距、表格字号和示例密度是否统一。
- 移动 375px：页面是否产生横向溢出，标题和按钮是否换行合理。
- 表格：表头字号、单元格字号、代码列等宽字体、行高和列对齐是否一致。
- 页面节奏：章节间距是否统一，说明文字是否比标题弱一层。
- Figma 同步前：页面状态已确认 stable，tokens 和 manifest 没有未同步差异。

## 推荐给 Codex 的任务描述

```text
按视觉一致性清单检查「布局页」：
只修标题标记、说明表格字号、卡片间距和移动端溢出；
不改文案结构，不新增组件 API；
改完运行 npm run verify 并给我 1440px 和 375px 截图。
```

## 不推荐的任务描述

```text
帮我全盘检查一下有没有问题。
```

这类描述范围太大，Codex 容易只检查代码是否通过，而漏掉设计师关心的视觉细节。
