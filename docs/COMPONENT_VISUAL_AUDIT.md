# 组件页视觉审计记录

审计日期：2026-06-25

本文件记录 29 个组件页在网页规范站中的视觉审计进度。当前阶段只确认规范站与前端交付链路，不进行 Figma 组件同步。

## 审计边界

- 视口：桌面 1440px、移动 375px。
- 检查：页面是否可访问、是否空白、页面级横向溢出、章节结构、说明型表格、移动端可读性。
- 暂不做：Figma 组件写入、Figma 组件状态发布、设计负责人最终视觉签核。

## 本轮结论

- 31 个组件页均可访问。
- 31 个组件页在桌面和 375px 移动端均无页面级横向溢出。
- 说明型表格已由 `DocsTable` 统一治理；业务组件自身的 Table 示例不计入说明型表格。
- Avatar、Badge、Tooltip、Popover 的浏览器文本溢出告警来自徽标/浮层绝对定位演示，未造成页面级横向溢出；后续做人工视觉签核时继续重点看这四页的局部示例。
- Figma 组件同步继续暂停；只有页面视觉签核完成后，才允许把组件状态推进到 Figma 同步。

## Figma 同步冻结规则

在所有组件页完成视觉签核前：

- 不调用 Figma 写入工具。
- 不把组件 `figma.syncStatus` 改为 `synced`。
- 只允许维护 `figma/tokens.json`、`figma/components.manifest.json` 作为本地合同与同步候选清单。

`npm run verify:visual` 会检查组件 manifest 中是否出现 `syncStatus: synced`；如果出现，会阻止验证通过。
