# 新材道设计规范变更记录

## 0.2.1 — 2026-06-23

- 完成剩余 24 个组件的规范收口，29 个组件统一进入 `stable`。
- 规范站为所有组件增加共享“完整交付合同”，统一展示结构、使用边界、交互、内容、响应式、可访问性、变体、状态及四类示例门槛。
- 为剩余组件补齐专属业务规则，移除通用占位式合同；同步更新 manifest、29 份 Markdown 与 Skill 结构化资产。
- Figma 状态仅调整为 `eligible`，本轮不写入 Figma，后续仍按批次集中同步。
- Button 新增 canonical `tone="task"`，黑色用于提交、确认、发布、创建等任务推进；默认 tone 从 neutral 调整为 task。
- 产品蓝明确为分析、生成、连接、筛选、导出等产品能力语义；能力型页面允许 product 成为唯一 solid。
- neutral 调整为取消、返回、关闭等中性操作；`neutral + solid` 保留一个迁移周期但不进入新示例。
- 颜色页和按钮页新增黑蓝决策顺序、场景矩阵与表单、分析、弹窗组合示例。
- 颜色页收窄决策卡并扩展场景矩阵，重构语义色为 2×2 状态卡，移除重复的快速选色指南。
- 页头移除面向维护者的“规范范围 / 真相源”信息，保留版本与成熟度。
- 成功、警告、错误文字色调整为更深色阶，改善白底与浅色背景上的可读性。
- 全局审计语义色引用，移除复制反馈与按钮文档中的旧语义色硬编码。
- 数据色规范结合 DBS Data Visualisation Guideline 重构为“是否需要图表 → 数据关系 → 图表类型 → 色彩与标注 → 边界检查”的决策流程。
- 明确状态语义色与图表数据色互不替代；单系列柱状图统一使用单色，并补充图表选型矩阵。
- 数据色重新建立为“产品蓝锚点 + 6 个核心分类色 + 6 个扩展色”，明确 12 色是上限而非常规用量；不直接复制 DBS 品牌色值。
- 图表示例移除 SVG 复制，扩充为折线、环图、柱状、堆叠柱、直方和发散条形六类，并统一 Caption 12/18 坐标标注。
- 修复环图各段间隔不一致，移除重复的图表结构与上线检查卡片。
- 语义实心按钮新增鲜明填充 Token，深色文字 Token 继续用于浅背景；按钮文案按 task、product、brand、neutral、danger 重新校对。
- 按钮字号、行高和移动触控目标重新对齐字体与交互基础规范。
- Textarea 对齐 80/96/120px 高度 Token，补齐字数统计、帮助信息关联、正确错误示例及 React/Vue 合同，状态升级为 stable。
- Select 收敛为可真实交付的原生单选合同，移除未实现的复合多选示例，补齐加载、错误关联、键盘规则和移动端标签回退，状态升级为 stable。

## 0.2.0 — 2026-06-22

- 固定正式范围为 29 个组件 + 7 个基础规范。
- 建立 Token 与组件 manifest 的真相源层级。
- 新增组件尺寸、焦点、动效、遮罩、触控目标和响应式 Token。
- 侧栏按基础规范、操作与输入、数据与内容、导航与组织、反馈与浮层重新分组。
- 统一规范页版本、成熟度和来源信息。
- 启动 `variant` / `tone` 语义迁移，并保留兼容别名。
- 组件 Markdown 改为从 manifest 生成定位、结构、合同、响应式、可访问性和示例要求。
- 新增 `draft → review → stable` 成熟度门槛，只有 stable 资产允许同步 Figma。
- 将 Figma v2.0（`KjkKSAd9eufpg9eFR9xZVX`）设为唯一正式目标，旧文件降级为历史参考。
- Button 完成 Token、React、Vue、文档及桌面/移动验收，成为首个 stable 组件。
- 7 个 Foundations 完成桌面与 375px 移动验收，解锁首次 Figma Foundations 同步。
- Figma v2.0 已建立 4 个单模式集合、222 个变量、10 个文字样式和 6 个阴影样式，进入 Foundations 人工验收检查点。
- 按用户决定暂停后续 Figma 写入，将额度集中到规范细节完善。
- Icon 统一 decorative 与 redMark 语义，补齐 React/Vue 可访问性合同并进入 stable。
- Input 统一 prefix/suffix 合同、错误说明关联、移动端 44px 控件高度与响应式标签布局，并进入 stable。
- 颜色页合并重复的色彩角色、语义和行动色章节，改为单一语义色合同；色阶使用渐变预览与变量表格呈现，数据色补充分类、连续和发散三种模型。
- 新增文字语义 Token，neutral-500 限制为禁用或非必要信息，规范站正文、描述、目录和标注提升至可读灰阶。
- 字体页减少说明文字堆积，改为官网内容页、后台详情页和数据看板等真实页面示例，并重构行高与可读性演示。
