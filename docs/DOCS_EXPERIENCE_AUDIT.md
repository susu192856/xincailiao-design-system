# 规范站交互、代码与角色语言审计

审计日期：2026-07-15

## 结论

- 组件不是越“能点”越好。会改变状态、选择、数据、导航或浮层的组件必须有真实交互；纯展示组件保留高质量视觉样本即可。
- 代码不是每页必需。交互组件在主要示例后放最小实现代码并默认收起；颜色、字体、间距等基础规范页不增加代码噪音。
- 正文统一使用共同语言说明对象、场景和结果。角色专属内容集中到明确标注的“设计与产品”或“开发与维护”区域。
- 文字密集模块优先用结构图、状态样本或真实业务组合解释；图不能替代必要的规则和可访问文字。

## 自动验收

`npm run verify:experience` 检查：

1. 24 个高交互页面是否包含真实交互证据。
2. 高交互页面是否包含默认收起的实现代码，且位置早于页尾规则。
3. 组件页标题区是否保持面向所有角色的共同语言。
4. 7 个纯视觉组件页是否包含真实渲染样本。

浏览器仍需人工验证打开、关闭、选择、输入、键盘焦点、窄宽度和页面溢出；自动脚本不能替代真实页面测试。
